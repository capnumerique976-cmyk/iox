## Mécanisme de déploiement (cause structurelle)

Lecture de `deploy/vps/deploy.sh` et `.github/workflows/ci.yml` :

- `ci.yml` : pas de step `ssh`, `rsync`, `vps`, `deploy` — **aucun auto-deploy**.
- `deploy/vps/deploy.sh` : script **manuel** (rsync + `docker compose build/restart`), à lancer depuis un poste avec accès SSH `IOX_VPS_HOST`.
- Cible par défaut : `frontend` ; `all` ou `backend` à passer en argument explicite.

→ Les 3 merges récents (PR #5/#6/#7) **n'ont pas pu déclencher de redéploiement**, c'est by-design. Il faut lancer manuellement `./deploy/vps/deploy.sh all` (ou `frontend` puis `backend`) pour propager.

## Run du script

```bash
TMP_ENV=$(mktemp); chmod 600 "$TMP_ENV"
sed 's|http://|https://|' ~/.iox-smoke.env > "$TMP_ENV"
SMOKE_ENV_FILE="$TMP_ENV" SMOKE_TIMEOUT=30 bash scripts/smoke-authenticated.sh
```

> Pourquoi le `sed` : `~/.iox-smoke.env` force `BASE_URL=http://iox.mycloud.yt` qui est sourcé **à l'intérieur** du script ; nginx renvoie un 301 → https que `curl` ne suit pas. Workaround sans toucher à l'env file utilisateur.

## Hypothèses sur la cause du drift

1. **Cause primaire** — pas de pipeline auto-deploy. `ci.yml` ne déploie pas. Tout merge sur `main` reste sur GitHub jusqu'à exécution manuelle de `deploy/vps/deploy.sh`.
2. **État backend incohérent** — l'image backend en prod a les routes `seller-profiles/me` (FP-3) et `media-assets/:id/url` (FP-3.1) mais PAS `certifications` (FP-2). Possibilités :
   - L'image actuelle a été buildée à partir d'un commit où `MarketplaceCertificationsModule` n'était pas encore importé dans `AppModule` (ou avait été retiré temporairement).
   - Le module est dans le code mais le `controller` n'est pas exporté → vérifier `apps/backend/src/marketplace-*/...module.ts` sur le commit déployé.
3. **Frontend** — pas de rebuild depuis avant les 5 merges. Le `buildId` `pIHPGlHix0hzwpn-vqAJh` est antérieur à toutes les pages introduites par FP-2.1 / FP-3.1 / FP-6. Donc côté UI, **aucune** des nouvelles pages (édition certifications, uploader inline, section "Origine détaillée") n'est encore visible en prod.
